Description

The next step is to update the recursive predicting function. If the feature is numeric, compare the values for their inequality rather than equality. For a numeric feature, a sample goes to the left child node if the value of this feature is less or equal to the threshold value. If otherwise, it should go to the right child node.

For categorical features, the algorithm remains the same.

Objectives

Firstly, incorporate the one-node-split function from the previous stage into your DecisionTree class. You may need to add a new DecisionTree class parameter: a list with names of numeric features. Ensure that the recursive splitting function works properly with the modified one-node-split function. Secondly, update the recursive predicting method.

  1. Take the input paths to two files split by a space: the first one is the training dataset, and the second one is the test dataset;
  2. Read the data from the .csv files
  3. There are two numerical features: Age and Fare;
  4. Train a decision tree on the training set. The minimum number of samples required to be at a leaf node is 1 by default. The Survived column is the target variable. Separate it from the dataset with features;
  5. Make predictions on the test set. Don't forget to separate the Survived feature in the test set as well.

The result of the stage is a set of log messages during the training process and prediction.

If you have corrupted CSV files, please download training dataset and test dataset. Place them in the test directory.
